Rows

The number of columns a grid has is specified by the columns attribute. For instance, to define a grid with 8 columns, you'd write <grid columns=8>. Max supported number of columns = 16. If you need more, you need to add them yourself.

 
<grid columns=8>
  <c span=3>Skelet</c>
  <c span=7-8>by</c>
  <c span=2+2>SÄ“lekkt.</c>
</grid>

Columns

The span attribute is used to customize where a cell starts and how many columns it spans.

For example, <c span=2-5> starts in column 2 and ends in column 5. A cell with an empty or missing span attribute starts on the next available column and spans one column. Cells span a single column by default.

span=value 2-5 start in column 2, end in column 5. 2+3 start in column 2, span 3 columns. 2.. start in column 2, span remainder of row. 2 start in next column, span 2 columns. row span a full row

Responsive

Grids lend themselves really well to device-size responsive design. The span-s attribute defines alternate layout when your design is presented on a small screen. This makes designing for large and small screens really easy.

<grid columns=6>
  <c span-l=4 span=2 span-s=row>
  <c span-l=1-2 span=3-6 span-s=row>
  <c span-l=3-6 span=1-2 span-s=1>
</grid>

Visual order

Specifies the order used to lay out GRID items in their GRID container. Elements are laid out in the ascending order of the order value. Elements with the same order value are laid out in the order in which they appear in the source code. order=*

Note: Since by default all CSS elements have an order: 0;, if you use order=* you need to spicify the order of every element (<c order=2>) otherwise the spicified element ends up last. [read more]

Available options
  • order=first
  • from order=1
  • to order=16
  • order=last

  • order-s=first
  • order-s=...
  • order-s=last
  •  
  • order-l=first
  • order-l=...
  • order-l=last
